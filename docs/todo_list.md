FlowCut TODO 리스트 (2026-04-22)
Part A. 기존 기능 보완
이미 구현된 기능의 안정성, 품질, 완성도를 높이는 작업들입니다.

A1. 세계관 일관성 시스템 보강
IPAdapter 참조 이미지 보강: 현재 시드 고정 방식은 프롬프트 변화나 모델 변경 시 캐릭터 일관성이 깨질 수 있음. IPAdapter(이미 설치됨)를 활용하여 첫 씬의 참조 얼굴을 이후 씬에 전파하는 이중 안전장치 구현
복장 변형 시 일관성 테스트: World Context에 복장 변형(variants)이 정의되어 있지만, 변형 간 전환 시 캐릭터 동일성이 실제로 유지되는지 체계적 검증 필요
다중 캐릭터 동시 등장 시 혼선 방지: 한 씬에 유나+민호가 동시에 등장할 때 외모/복장이 뒤바뀌는 문제 대응 전략 수립
A2. AI 생성 품질 향상
업스케일 파이프라인 고도화: 현재 FFmpeg Lanczos 필터는 단순 보간에 불과. RealESRGAN 또는 SwinIR 등 AI 업스케일러를 ComfyUI 워크플로우에 통합하여 네이티브에 가까운 1080p 출력 달성
I2V 프레임 보간 적용: 81프레임/5초(16fps)는 영상으로서 부드러움이 부족. comfyui-frame-interpolation(이미 설치됨)을 파이프라인에 적용하여 30fps 이상으로 끌어올리기
프롬프트 강화 시 과적합 방지: 시네마틱 강화가 42종 카메라 + 조명 + 샷 사이즈를 자동 주입하는데, 프롬프트가 과도하게 길어지면 모델이 핵심 내용을 놓칠 수 있음. 토큰 수 제한 또는 우선순위 기반 프롬프트 트리밍 로직 추가
A3. TTS 오디오 안정성
TTS 타이밍 정밀 동기화: 생성된 음성의 길이와 씬 클립 길이 간 불일치 시 자동 조절(음성이 길면 씬 연장 or 음성 속도 조절) 로직 보완
음성 품질 일관성: Edge TTS의 보이스별 품질 편차가 존재. 보이스 프리셋별 권장 설정(속도, 피치, 볼륨) 기본값 튜닝
긴 텍스트 분할 처리: 한 씬의 나레이션이 길 경우 TTS가 끊기거나 품질이 떨어지는 문제 대응. 문장 단위 분할 생성 후 결합 방식 적용
A4. NLE 에디터 안정화
비디오 프리뷰 성능 최적화: 다수 비디오 클립이 타임라인에 올라가면 프리뷰 캔버스 렌더링 부하 증가. 비활성 클립의 비디오 디코딩 해제, 썸네일 캐싱 등 메모리 관리 강화
크로스페이드 전환 프리뷰: buildXfadeFilter는 FFmpeg Export 시에만 적용됨. 프리뷰 캔버스에서도 전환 효과를 실시간으로 확인할 수 있도록 WebGL 기반 전환 렌더링 구현
프로젝트 저장/로드 안정성: ProjectManager의 auto video-type detection이 구현되었지만, 대규모 프로젝트(20+ 트랙, 100+ 클립) 시 로드 성능과 데이터 무결성 검증 필요
FlowScript 액션 에러 핸들링: 27개 액션 중 실패 시 롤백 처리가 미비한 액션 점검. 특히 AI 생성 액션(generateImage, generateVideo, generateTTS)은 외부 서비스 의존이므로 타임아웃/재시도/폴백 로직 보강
A5. 개발 인프라
자동화 테스트 도입: 현재 Console 수동 검증에서 벗어나 FlowScript 액션별 단위 테스트, 파이프라인 통합 테스트(Vitest/Playwright) 구축
에러 로깅 체계화: 서버(Express) + 클라이언트(React) 양쪽의 에러를 구조화된 로그로 수집. AI 생성 실패 원인 추적을 위한 ComfyUI 응답 로깅
Part B. 신규 기능 개발
아직 구현되지 않은 새로운 기능들입니다.

B1. P2 Week 3 — 영상 확장 기능
무한 비디오 체이닝: 5초 클립 제한을 넘어서 앞 클립의 마지막 프레임을 다음 클립의 시작 이미지로 사용하여 연속적인 장면 생성. 모션과 구도의 연속성 유지 알고리즘 설계
모션 라이브러리: 42종 카메라 무브먼트를 사용자가 직접 선택/조합할 수 있는 UI. 드래그 앤 드롭으로 씬에 모션 프리셋 적용, 커스텀 모션 저장
프로젝트 템플릿 시스템: YouTube Shorts(9:16, 30초, 15씬), Instagram Reels(9:16, 60초, 30씬), YouTube Video(16:9, 3분, 챕터), 광고 크리에이티브(15/30/60초) 등 사전 정의 템플릿. 커스텀 템플릿 빌더로 사용자가 직접 beat 구조 설계 가능
B2. P2 Week 4 — 플랫폼 확장
클라우드 Provider 실제 구현: Open Architecture의 Provider Interface가 설계되었으니, fal.ai/Replicate/RunPod 등 최소 1개 클라우드 Provider를 실제 구현하여 로컬 GPU 없이도 사용 가능하게 만들기
배치 Export: 동일 프로젝트를 여러 화면비(16:9 + 9:16 + 1:1)로 동시 내보내기. A/B 배리언트 생성(다른 프롬프트, 같은 구조)으로 콘텐츠 테스트 지원
통합 테스트 스위트: 전체 파이프라인(프롬프트 → 씬 생성 → 이미지 → 비디오 → Export) end-to-end 테스트 자동화
B3. 오디오 완성
BGM 생성/라이브러리: MusicGen 또는 로컬 음악 생성 모델 통합. 씬 분위기(긴장/따뜻함/액션)에 맞는 BGM 자동 선택/생성
볼륨 덕킹 자동화: 나레이션 구간에서 BGM 볼륨을 자동으로 낮추고 나레이션 끝나면 복원. 덕킹 커브(linear/ease)와 감쇠량(dB) 설정 가능
SFX(효과음) 라이브러리: 전환 효과음, 강조 효과음 등 기본 SFX 세트 내장. 씬 유형별 자동 SFX 추천
B4. 사용자 경험
온보딩 위자드: 첫 실행 시 FFmpeg/ComfyUI/Ollama 설치 상태 자동 감지 및 가이드. 클라우드 Provider 선택 시 로컬 설치 불필요 경로 안내
프롬프트 어시스턴트 UI: World Context와 캐릭터 레지스트리를 코드/JSON이 아닌 시각적 폼으로 편집할 수 있는 패널. 캐릭터 카드(썸네일+이름+시드+설명) 형태의 관리 UI
생성 진행률 대시보드: AI 파이프라인 각 단계(LLM → 이미지 → i2v → 업스케일 → TTS)의 진행 상태를 실시간으로 보여주는 UI. 예상 소요 시간 표시, 단계별 재시도 버튼
B5. P3 장기 기능
립싱크/아바타: TTS 음성에 맞춰 캐릭터 입 모양을 동기화하는 디지털 아바타 기능. HeyGen/Synthesia 대비 로컬+무료 포지셔닝
실시간 협업: 다중 사용자가 동시에 프로젝트를 편집할 수 있는 WebSocket 기반 협업 시스템. 공유 에셋 라이브러리
플러그인 시스템: 커스텀 ComfyUI 워크플로우 임포트, 서드파티 이펙트 플러그인, 커뮤니티 템플릿 마켓플레이스
클라우드 GPU 렌더팜: 로컬 편집 + 클라우드 생성 하이브리드 모드. 렌더 큐 관리, 비용 추적 대시보드
